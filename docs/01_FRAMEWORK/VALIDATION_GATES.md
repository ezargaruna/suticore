# validation_gates

## purpose
- prevent noise
- stop premature output
- keep system honest

no gate → no output.

---

## gate 1 · clarity
question:
- is there one node?

checks:
- one core topic
- no secondary themes
- no mixed intentions

fail conditions:
- multiple problems
- broad framing
- unclear request

if fail:
↺ reduce  
or ∅ silence

---

## gate 2 · silence
question:
- does the text breathe?

checks:
- no urgency markers
- no emotional spikes
- no persuasive rhythm

fail conditions:
- pressure language
- hype
- escalation tone

if fail:
⧖ pause  
reduce output

---

## gate 3 · step
question:
- is there one step?

checks:
- exactly one ⟁
- ≤10 minutes
- possible today
- reversible

fail conditions:
- no action
- many actions
- abstract tasks

if fail:
↺ rebuild  
or ∅ silence

---

## gate 4 · integrity
question:
- is form preserved?

checks:
- format canon respected
- symbols used functionally
- limits not exceeded

fail conditions:
- decorative symbols
- long lines
- nested explanations

if fail:
↺ reformat

---

## gate 5 · ethics
question:
- is choice preserved?

checks:
- no manipulation
- no diagnosis
- no promises
- no authority claims

fail conditions:
- advice framing
- outcome guarantees
- hidden persuasion

if fail:
∅ silence

---

## escalation rule
if more than one gate fails:
- stop processing
- no rewrite attempt
- silence is output

---

## review posture
- slow
- non-reactive
- minimal corrections

do not optimize.  
do not expand.

---

## scope
these gates apply to:
- public docs
- examples
- ai responses
- comments
- metadata

private layers may diverge.

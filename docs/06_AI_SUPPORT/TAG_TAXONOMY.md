# tag_taxonomy

## purpose
- reduce search noise
- support orientation
- enable quiet filtering

tags are signals.  
not labels.

---

## tag principles
- low quantity
- high precision
- non-promotional
- non-exhaustive

tags do not explain.  
they point.

---

## tag layers

### state tags
describe internal state  
the content is tuned for

examples:
- state:pause
- state:clarity
- state:fatigue
- state:threshold
- state:orientation

rules:
- one state tag max
- never mixed

---

### format tags
describe structural form  
not topic

examples:
- format:canon
- format:framework
- format:protocol
- format:reference
- format:example

rules:
- exactly one format tag
- mandatory for all docs

---

### action tags
describe expected movement  
after reading

examples:
- action:pause
- action:step
- action:return
- action:stop

rules:
- optional
- never more than one

---

### season tags
describe temporal context  
not urgency

examples:
- season:dark
- season:transition
- season:integration

rules:
- optional
- used sparingly

---

## forbidden tag types
- seo keywords
- emotional tags
- outcome tags
- marketing language
- identity claims

examples forbidden:
- success
- productivity
- healing
- mastery
- growth

---

## tag placement
- top of document
- after title
- before body

format:
`tags: state:clarity · format:canon`

---

## tag reading posture
- tags set context
- tags do not promise
- tags do not rank

ignore tags →  
content still valid.

---

## maintenance rules
- review quarterly
- remove unused tags
- no expansion without need

taxonomy must stay  
small and legible.

---

## scope
applies to:
- docs/*
- examples
- comments (if tagged)
- weekly exports

private layers may diverge.

---

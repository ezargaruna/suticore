# decision_stack

## purpose
- prevent impulsive output
- keep one vector
- ensure action exists

## applies always
- any request
- any content
- any decision
- any response

## step 1 · ∴ node
- extract one core question
- remove secondary themes
- name the decision point

## step 2 · ⧖ friction
- check internal resistance
- check external pressure
- separate fact from assumption

## step 3 · ⟁ step
- one concrete action
- ≤10 minutes
- possible today
- reversible

## step 4 · 〄 closure
- fix the decision
- mark next review
- no further expansion

## validation rules
- if no step → incomplete
- if many steps → reduce
- if pressure → ⧖ pause
- if unclear → ↺ return

## option handling
- default: one step
- if choice required:
  ➤ offer 3 options max
- no ranking
- no persuasion

## failure handling
- stop is valid
- silence is valid
- rollback is allowed

## output constraint
- one vector only
- no parallel advice
- no hidden agendas

## status
- active canon
- version: 2026.2
